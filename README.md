# Bay Club Court Checker

A multi-agent Python tool that autonomously builds itself via a Planner → Generator → Evaluator loop, then checks Bay Club tennis court availability and notifies you when a slot opens.

## How it works

You provide one prompt. The orchestrator does the rest:

1. **Planner** — turns your prompt into a spec and sprint breakdown
2. **Generator** — builds the app sprint by sprint
3. **Evaluator** — tests each sprint; loops back if it fails

Each agent is a separate `claude -p` subprocess. They share no memory — only files in `workspace/`.

## Usage

### Prerequisites

```bash
pip install -r requirements.txt  # generated by the harness
```

Set credentials in your environment:

```bash
export BAY_CLUB_USERNAME="your@email.com"
export BAY_CLUB_PASSWORD="yourpassword"

# Optional notification settings
export NOTIFY_EMAIL="your@email.com"       # for email alerts
export SMTP_HOST="smtp.gmail.com"
export SMTP_PORT="587"
export SMTP_USER="sender@gmail.com"
export SMTP_PASSWORD="app-password"
```

### Run the harness

```bash
python orchestrator.py "build me a Bay Club tennis court availability checker"
```

The harness plans, builds, and evaluates autonomously. Re-running resumes from the last completed sprint.

### Run the checker (after the harness completes)

The generated app will be in `workspace/implementation/`. See `workspace/spec.md` for exact usage, but typically:

```bash
python workspace/implementation/checker.py \
  --location "SF" \
  --court-type "tennis" \
  --date "2026-04-05" \
  --time-start "07:00" \
  --time-end "10:00" \
  --interval 300   # check every 5 minutes
```

## Configuration

| Variable | Description | Default |
|---|---|---|
| `BAY_CLUB_USERNAME` | Member portal login email | required |
| `BAY_CLUB_PASSWORD` | Member portal password | required |
| `BAY_CLUB_LOCATION` | Club location filter (e.g. "SF", "Redwood Shores") | all |
| `NOTIFY_EMAIL` | Email to send alerts to | desktop notification |
| `CHECK_INTERVAL` | Seconds between checks | 300 |

## Architecture

```
workspace/
├── spec.md                  ← Planner writes this
├── sprints.json             ← Planner's sprint breakdown
├── implementation/          ← Generator builds here
├── build_summary.md         ← Generator's self-review
├── evaluation_report.json   ← Evaluator's verdict
└── state.json               ← Orchestrator resumability
```

## Resumability

If the orchestrator is interrupted, re-run the same command. It reads `workspace/state.json` and skips completed sprints.

To force a full restart: `rm workspace/state.json`

## Cost

Each sprint costs ~$1–5 in Claude API usage depending on complexity. The harness has a configurable `MAX_COST_USD` guard (default: $50).
